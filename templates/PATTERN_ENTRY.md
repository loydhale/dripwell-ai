# Pattern entry template

Use this format when appending to `memory/PATTERNS.md`:

```
## P-<next-number> — <short-name>
Date: <YYYY-MM-DD>
Where in codebase: <file paths or module>
The pattern: <description of the approach>
When to use: <conditions>
Example: <pointer to reference implementation>
```

Rules:
- Coders must follow documented patterns even over personal preference
- Auditor enforces during review
- Delete if the pattern is no longer present in the codebase
